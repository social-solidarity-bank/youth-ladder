# 청년 희망사다리 프로젝트 Brief

## 요청 핵심

- 온라인 신청서 접수 페이지와 관리자 모니터링 페이지의 기본 작동 상태를 먼저 확인한다.
- 확인된 오류는 운영 전 위험이 큰 항목부터 바로 정리한다.
- 다음 단계로 관리자 화면 안에서 사용할 심사 화면을 설계한다.

## 이해관계자

- 신청자: 모바일 또는 PC에서 신청서를 작성하고 제출하는 청년
- 관리자: 접수 목록을 확인하고 신청서를 열람하는 실무자
- 심사자: 자격, 부채, 소득, 신청 사유를 검토하고 판정을 남기는 담당자
- 사업 관리자: 심사 현황, 보완 요청, 최종 선정 흐름을 모니터링하는 담당자

## 금기 및 제약

- 주민등록번호, 연락처, 주소, 소득, 채무 등 민감정보를 브라우저 저장소에 남기지 않는다.
- 신청서 제출 전 필수 항목 검증 없이 서버로 전송하지 않는다.
- 관리자 화면은 Supabase Auth 기반 로그인 이후에만 신청 정보를 조회한다.
- 심사 판단 기록은 누가, 언제, 어떤 근거로 변경했는지 추적 가능해야 한다.

## 현재 확인 결과

- 접수 페이지: 자격확인, 동의, 신청서 진입까지 브라우저에서 정상 작동 확인.
- 관리자 페이지: 로그인 화면 로딩과 빈 로그인 검증 메시지 정상 확인.
- 제한 사항: 실제 관리자 신청 목록 조회는 계정 세션이 필요해 로그인 이후 데이터 조회까지는 미확인.
- 수정 완료: 빈 신청서 제출 방지, 민감정보 자동저장 제거.
